# 凭据管理功能

## 📋 功能概述

凭据管理功能允许用户创建和管理可复用的数据库账号密码,在创建数据源时可以选择使用已有凭据,避免重复输入相同的账号密码。

**适用场景**:
- 测试环境多个数据库使用相同的账号密码
- 生产环境统一管理数据库凭据
- 需要批量更新多个数据源的密码

---

## 🎯 核心功能

### 1. 凭据管理
- 创建凭据:保存账号密码并加密存储
- 编辑凭据:修改凭据信息(密码可选更新)
- 删除凭据:删除不再使用的凭据(被数据源使用时无法删除)
- 查看凭据列表:展示所有可用凭据

### 2. 数据源集成
- 创建数据源时可选择使用凭据或手动输入
- 使用凭据的数据源会自动从凭据获取账号密码
- 支持在编辑时切换认证方式

---

## 📊 数据库设计

### credentials 表

```sql
CREATE TABLE credentials (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,      -- 凭据名称(唯一)
    description TEXT,                -- 描述
    username TEXT NOT NULL,          -- 用户名
    password TEXT NOT NULL,          -- 加密存储的密码
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### data_sources 表修改

```sql
ALTER TABLE data_sources ADD COLUMN credential_id TEXT;
-- credential_id 为空时使用自己的 username/password
-- credential_id 不为空时从凭据获取账号密码
```

## 🔄 工作流程

### 创建凭据
1. 进入"凭据管理"页面
2. 点击"新建凭据"
3. 填写凭据名称、用户名、密码、描述
4. 保存后密码会被AES-256-GCM加密存储

### 使用凭据创建数据源
1. 进入"数据源配置"页面
2. 点击"新建数据源"
3. 选择"认证方式"为"使用凭据"
4. 从下拉列表选择已创建的凭据
5. 填写其他信息后保存

### 手动输入创建数据源
1. 选择"认证方式"为"手动输入"
2. 直接输入用户名和密码
3. 保存后密码会被加密存储

---

## 🔒 安全性

### 密码加密
- 使用AES-256-GCM加密算法
- 密码在数据库中以密文存储
- 只在需要连接时才解密

### 权限控制
- 凭据被数据源使用时无法删除
- 删除凭据前会检查关联的数据源数量

---

## 💡 使用建议

### 最佳实践
1. 为不同环境创建不同的凭据(如:测试环境凭据、生产环境凭据)
2. 使用描述字段说明凭据的用途和适用范围
3. 定期更新凭据密码以提高安全性
4. 删除不再使用的凭据

### 注意事项
1. 凭据名称必须唯一
2. 删除凭据前需要先解除所有数据源的关联
3. 编辑凭据时如果不修改密码可以留空
4. 使用凭据的数据源会自动使用凭据的最新密码

## 🔧 API接口

### 凭据管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/credentials | 获取凭据列表 |
| POST | /api/v1/credentials | 创建凭据 |
| GET | /api/v1/credentials/:id | 获取凭据详情 |
| PUT | /api/v1/credentials/:id | 更新凭据 |
| DELETE | /api/v1/credentials/:id | 删除凭据 |

### 请求示例

**创建凭据**
```json
POST /api/v1/credentials
{
  "name": "测试环境凭据",
  "username": "root",
  "password": "password123",
  "description": "测试环境MySQL统一凭据"
}
```

**创建使用凭据的数据源**
```json
POST /api/v1/datasources
{
  "name": "测试数据库1",
  "type": "mysql",
  "host": "localhost",
  "port": 3306,
  "credential_id": "credential-uuid-here",
  "database_name": "test_db"
}
```
