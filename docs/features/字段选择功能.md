# 字段选择功能

## 功能说明

允许用户在配置同步任务时，选择表的部分字段进行同步，而不是同步所有字段。

## 使用场景

- 源表有很多字段，但只需要同步部分字段
- 减少网络传输和存储空间
- 提高同步效率

## 使用方法

1. 在任务配置向导的"选择表"步骤
2. 点击表旁边的"字段"按钮
3. 勾选需要同步的字段
4. 点击"确定"保存

## 注意事项

- 如果不选择任何字段，默认同步所有字段
- 字段选择后会显示已选字段数量
- 全量和增量同步都支持字段过滤

## 工作原理

### 配置阶段

1. 用户点击"字段"按钮
2. 系统查询数据库获取表的所有字段
3. 显示字段列表供用户勾选
4. 用户选择的字段保存到任务配置（MySQL）

### 执行阶段

**全量同步：**
```sql
-- 如果选择了字段
SELECT id, name, email FROM users

-- 如果没有选择字段（默认）
SELECT * FROM users
```

**增量同步：**
- Binlog事件中只保留选中的字段
- INSERT/UPDATE/DELETE都会过滤字段

## API接口

### 获取表字段列表

**请求：**
```
GET /api/v1/datasources/{id}/tables/{database}/{table}/columns
```

**参数：**
- `id`: 数据源ID
- `database`: 数据库名（需要URL编码）
- `table`: 表名（需要URL编码）

**响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": ["id", "name", "age", "email", "created_at"]
}
```

## 配置存储

用户选择的字段保存在 `sync_tasks` 表的 `config` 字段中：

```json
{
  "selected_databases": [
    {
      "database": "test_db",
      "tables": [
        {
          "source_table": "users",
          "target_table": "users",
          "selected_fields": ["id", "name", "email"]
        }
      ]
    }
  ]
}
```

## 调试方法

### 前端调试

1. 打开浏览器开发者工具（F12）
2. 切换到Console标签
3. 点击"字段"按钮
4. 查看控制台输出的URL和错误信息

### 后端调试

1. 查看后端日志，确认请求是否到达
2. 测试数据库连接
3. 确认表名和数据库名是否正确

### 常见问题

**问题1：点击字段按钮没反应**
- 检查浏览器控制台是否有错误
- 确认数据源ID是否正确
- 检查网络请求是否发送

**问题2：提示"获取字段列表失败"**
- 检查数据库名和表名是否包含特殊字符
- 确认数据源连接是否正常
- 查看后端日志的详细错误信息

**问题3：字段列表为空**
- 确认表是否存在
- 确认表是否有字段
- 检查数据库连接权限

## 性能说明

- 查询字段列表：50-200ms（取决于表的字段数量）
- 配置不频繁，性能影响可忽略
- 直接查询数据库，数据永远是最新的
