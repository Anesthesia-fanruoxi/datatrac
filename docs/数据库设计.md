# 数据库设计

> 更新时间：2024年（重构后）
> 
> 本文档说明重构后的数据库设计，已删除进度表和配置表

## 📊 表结构概览

系统使用 **MySQL** 存储核心配置，使用 **Redis** 存储详情配置和统计数据。

### MySQL 表（2个）

1. `data_sources` - 数据源表
2. `sync_tasks` - 同步任务表

### Redis 数据（2类）

1. `task:config:{taskID}` - 任务详细配置
2. `task:stats:{taskID}` - 增量同步统计

---

## 📋 MySQL 表设计

### 1. data_sources - 数据源表

**用途**: 存储数据源连接信息

```sql
CREATE TABLE `data_sources` (
    `id` VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    `name` VARCHAR(100) NOT NULL COMMENT '数据源名称',
    `type` VARCHAR(20) NOT NULL COMMENT '数据源类型: mysql/elasticsearch',
    `host` VARCHAR(255) NOT NULL COMMENT '主机地址',
    `port` INT NOT NULL COMMENT '端口',
    `username` VARCHAR(100) NOT NULL COMMENT '用户名',
    `password` VARCHAR(255) NOT NULL COMMENT '密码（加密存储）',
    `database_name` VARCHAR(100) COMMENT '数据库名（MySQL专用）',
    `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX `idx_name` (`name`),
    INDEX `idx_type` (`type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='数据源表';
```

**字段说明**:
- `password`: 使用 AES-256-GCM 加密存储
- `database_name`: 仅 MySQL 类型使用

---

### 2. sync_tasks - 同步任务表

**用途**: 存储任务基本信息和最终状态

```sql
CREATE TABLE `sync_tasks` (
    `id` VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    `name` VARCHAR(100) NOT NULL COMMENT '任务名称',
    `source_id` VARCHAR(36) COMMENT '源数据源ID',
    `target_id` VARCHAR(36) COMMENT '目标数据源ID',
    `source_type` VARCHAR(20) NOT NULL COMMENT '源类型: mysql/elasticsearch',
    `target_type` VARCHAR(20) NOT NULL COMMENT '目标类型: mysql/elasticsearch',
    `config` TEXT NOT NULL COMMENT 'JSON格式的基本配置',
    `status` VARCHAR(20) NOT NULL DEFAULT 'idle' COMMENT '任务状态: idle/configured',
    `is_running` BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否正在运行',
    `sync_mode` VARCHAR(20) NOT NULL DEFAULT 'full' COMMENT '同步模式: full/incremental',
    `current_step` VARCHAR(50) DEFAULT '' COMMENT '当前步骤: initialize/sync_data/completed/incremental',
    `queue_type` VARCHAR(20) DEFAULT 'memory' COMMENT '队列类型: memory/redis',
    `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX `idx_status` (`status`),
    INDEX `idx_is_running` (`is_running`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='同步任务表';
```

**字段说明**:
- `config`: 存储基本配置的JSON，详细配置在Redis中
- `status`: 任务配置状态（idle未配置，configured已配置）
- `is_running`: 任务运行状态
- `current_step`: 当前执行步骤，用于前端展示
- `queue_type`: 增量同步使用的队列类型

**状态流转**:
```
idle (创建) → configured (配置完成) → is_running=true (运行中) → is_running=false (完成)
```

## 🔴 Redis 数据设计

### 1. 任务详细配置

**Key**: `task:config:{taskID}`

**类型**: String (JSON)

**内容**:
```json
{
  "source_id": "xxx",
  "target_id": "xxx",
  "selected_databases": [
    {
      "database": "target_db",
      "source_database": "source_db",
      "is_database_modified": false,
      "tables": [
        {
          "source_table": "users",
          "target_table": "users",
          "is_modified": false
        }
      ]
    }
  ],
  "sync_config": {
    "batch_size": 1000,
    "thread_count": 4,
    "sync_mode": "full",
    "error_strategy": "pause",
    "table_exists_strategy": "drop"
  }
}
```

**用途**:
- 存储表映射关系
- 存储字段映射关系
- 存储同步参数配置

**生命周期**:
- 任务配置时创建
- 任务配置修改时更新
- 任务删除时清除
- 服务启动时从MySQL重建

---

### 2. 增量同步统计

**Key**: `task:stats:{taskID}`

**类型**: Hash

**内容**:
```
Field: db1.table1
Value: {
  "insert_count": 100,
  "update_count": 50,
  "delete_count": 10,
  "last_update": "2024-01-01 12:00:00"
}

Field: db1.table2
Value: {...}
```

**用途**:
- 记录每个表的增量操作统计
- 用于前端展示增量同步进度

**生命周期**:
- 增量同步启动时初始化
- 增量消费时实时更新
- 任务停止时保留
- 任务删除时清除

## 💾 内存数据设计

### 运行时进度（全量同步）

**数据结构**:
```go
type TaskProgressData struct {
    TaskID      string
    Units       map[string]*TaskUnit  // unitName -> unit
    StartTime   time.Time
    CurrentStep string
}

type TaskUnit struct {
    UnitName         string    // 格式: database.table
    Status           string    // pending/running/completed/failed/initialized
    TotalRecords     int64
    ProcessedRecords int64
    ErrorMessage     string
    StartedAt        *time.Time
    UpdatedAt        time.Time
}
```

**用途**:
- 全量同步时维护实时进度
- 通过SSE推送给前端

**生命周期**:
- 任务启动时初始化
- 任务运行时实时更新
- 任务完成/停止时清空

## 🔄 数据流转

### 配置流转

```
前端提交配置
  ↓
保存到 MySQL (sync_tasks.config)
  ↓
解析并保存到 Redis (task:config:{taskID})
  ↓
任务启动时从 Redis 读取
```

### 进度流转（全量同步）

```
任务启动
  ↓
初始化内存进度 (TaskProgressManager)
  ↓
Worker 更新内存进度
  ↓
SSE 推送给前端
  ↓
任务完成
  ├─ 更新 MySQL (is_running, current_step)
  └─ 清空内存进度
```

### 统计流转（增量同步）

```
Binlog 事件
  ↓
增量消费处理
  ↓
更新 Redis 统计 (task:stats:{taskID})
  ↓
SSE 推送给前端
```

## 🎯 设计优势

### 1. 性能优化
- 全量同步无数据库IO，完全在内存
- 减少了MySQL的写入压力
- 提高了同步速度

### 2. 架构简化
- 删除了2个表，减少了表关联
- 代码更清晰，逻辑更简单
- 维护成本降低

### 3. 职责明确
- **MySQL**: 持久化核心配置和最终状态
- **Redis**: 缓存详情配置和增量统计
- **内存**: 临时进度数据

### 4. 灵活扩展
- Redis配置可以支持更复杂的映射关系
- 内存进度结构可以灵活调整
- 不受数据库表结构限制

---

## ⚠️ 注意事项

### 1. Redis 依赖
- 系统依赖 Redis 存储配置
- Redis 故障时可以从 MySQL 降级读取基本配置
- 建议配置 Redis 持久化（RDB + AOF）

### 2. 数据一致性
- 配置修改时先更新 MySQL，再更新 Redis
- 服务重启时从 MySQL 重建 Redis 配置
- 增量统计数据仅在 Redis，不持久化到 MySQL

### 3. 内存管理
- 任务完成后及时清理内存进度
- 避免内存泄漏
- 大量并发任务时注意内存使用
