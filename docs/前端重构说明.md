# 前端架构重构说明

## 重构目标

将原有的单页应用（SPA）架构重构为多页应用（MPA）架构，实现模块化和隔离，每个菜单独立页面和资源。

## 模块设计

### 1. 通用组件（Components）

所有模块共享的组件，提供基础功能：

- **Toast**: 消息通知组件
  - `Toast.success(message)` - 成功提示
  - `Toast.error(message)` - 错误提示
  - `Toast.warning(message)` - 警告提示
  - `Toast.info(message)` - 信息提示

- **Modal**: 对话框组件
  - `Modal.confirm(message, onConfirm, onCancel)` - 确认对话框

- **FormatUtils**: 格式化工具
  - `FormatUtils.formatDate(dateString)` - 格式化日期
  - `FormatUtils.formatSize(bytes)` - 格式化文件大小
  - `FormatUtils.formatNumber(num)` - 格式化数字
  - `FormatUtils.formatDuration(seconds)` - 格式化时间间隔

- **HttpUtils**: HTTP 请求封装
  - `HttpUtils.get(url)` - GET 请求
  - `HttpUtils.post(url, data)` - POST 请求
  - `HttpUtils.put(url, data)` - PUT 请求
  - `HttpUtils.delete(url)` - DELETE 请求

### 2. 业务模块（Modules）

每个模块包含三个文件：

- **index.js**: 模块入口文件
  - `init()` - 初始化模块
  - `destroy()` - 清理模块资源
  - `bindEvents()` - 绑定事件

- **[module].js**: 核心业务逻辑
  - 数据加载
  - 业务处理
  - UI 更新

- **[module].css**: 模块样式
  - 模块特定样式
  - 不影响其他模块

### 3. 页面模板（Templates）

每个页面独立的 HTML 文件：

- 引入公共布局（header、sidebar、footer）
- 引入模块样式
- 引入通用组件
- 引入模块脚本
- 调用模块初始化

## 路由配置

```go
// 页面路由
r.GET("/", pageCtrl.Index)                      // 首页（重定向到数据源）
r.GET("/datasources", pageCtrl.DataSource)     // 数据源页面
r.GET("/task-config", pageCtrl.TaskConfig)     // 任务配置页面
r.GET("/task-monitor", pageCtrl.TaskMonitor)   // 任务监控页面
r.GET("/task-monitor/:id", pageCtrl.TaskMonitorWithID) // 任务监控（带ID）
```

## 模块生命周期

### 初始化流程

1. 页面加载
2. 加载通用组件（Toast、Modal、Utils）
3. 加载模块脚本
4. 调用模块 `init()` 方法
5. 模块加载数据并渲染 UI

### 清理流程

1. 用户离开页面
2. 触发 `beforeunload` 事件
3. 调用模块 `destroy()` 方法
4. 关闭 SSE 连接
5. 清理事件监听器

## 优势

1. **模块隔离**: 每个模块独立，互不影响
2. **代码清晰**: 文件结构清晰，易于维护
3. **按需加载**: 只加载当前页面需要的资源
4. **易于扩展**: 新增模块只需创建对应目录和文件
5. **团队协作**: 不同开发者可以独立开发不同模块
