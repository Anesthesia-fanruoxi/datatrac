# DataTrace 用户权限要求

本文档说明 DataTrace 数据同步系统对源数据库用户权限的要求。

## 概述

DataTrace 使用**只读权限**连接源数据库，不会对源数据库进行任何修改操作，确保源数据库的安全性。

## 权限要求

### 1. 全量同步所需权限

全量同步需要读取源数据库的表结构和数据。

#### 基础权限

```sql
-- 查询数据权限（必需）
GRANT SELECT ON source_database.* TO 'datatrace_sync'@'%';

-- 查看视图定义权限（如果同步视图则必需）
GRANT SHOW VIEW ON source_database.* TO 'datatrace_sync'@'%';

-- 查询数据库列表权限（必需）
GRANT SHOW DATABASES ON *.* TO 'datatrace_sync'@'%';
```

### 2. 增量同步所需权限

增量同步通过读取 MySQL Binlog 实现实时数据同步。

#### Binlog 读取权限

```sql
-- 读取 binlog 事件（必需）
GRANT REPLICATION SLAVE ON *.* TO 'datatrace_sync'@'%';

-- 查看 binlog 状态（必需）
GRANT REPLICATION CLIENT ON *.* TO 'datatrace_sync'@'%';
```

## 完整配置示例

### 创建同步专用用户

```sql
-- 1. 创建用户
CREATE USER 'datatrace_sync'@'%' IDENTIFIED BY 'your_secure_password';

-- 2. 授予全量同步权限（只读）
GRANT SELECT, SHOW VIEW ON source_database.* TO 'datatrace_sync'@'%';

-- 3. 授予元数据查询权限
GRANT SHOW DATABASES ON *.* TO 'datatrace_sync'@'%';

-- 4. 授予增量同步权限（如果需要 binlog）
GRANT REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'datatrace_sync'@'%';

-- 5. 刷新权限
FLUSH PRIVILEGES;
```

### 多数据库授权

如果需要同步多个数据库：

```sql
-- 方式1：逐个授权（推荐）
GRANT SELECT, SHOW VIEW ON db1.* TO 'datatrace_sync'@'%';
GRANT SELECT, SHOW VIEW ON db2.* TO 'datatrace_sync'@'%';
GRANT SELECT, SHOW VIEW ON db3.* TO 'datatrace_sync'@'%';

-- 方式2：授权所有数据库（不推荐生产环境）
GRANT SELECT, SHOW VIEW ON *.* TO 'datatrace_sync'@'%';
```

## 权限说明

| 权限 | 用途 | 是否必需 | 是否影响源库 |
|------|------|----------|--------------|
| `SELECT` | 读取表数据 | ✅ 必需 | ❌ 只读，不影响 |
| `SHOW VIEW` | 查看视图定义 | ⚠️ 同步视图时必需 | ❌ 只读，不影响 |
| `SHOW DATABASES` | 列出数据库 | ✅ 必需 | ❌ 只读，不影响 |
| `REPLICATION SLAVE` | 读取 binlog | ⚠️ 增量同步必需 | ❌ 只读，不影响 |
| `REPLICATION CLIENT` | 查看 binlog 状态 | ⚠️ 增量同步必需 | ❌ 只读，不影响 |

## 源数据库配置要求

### 增量同步配置

如果使用增量同步，源数据库需要开启 binlog：

```ini
# my.cnf 或 my.ini
[mysqld]
# 开启 binlog
log-bin=mysql-bin

# binlog 格式（必须是 ROW）
binlog_format=ROW

# binlog 保留时间（天）
expire_logs_days=7

# 服务器 ID（必须唯一）
server-id=1
```

### 配置说明

- **log-bin**：开启 binlog 功能
- **binlog_format**：必须设置为 `ROW`，DataTrace 只支持 ROW 格式
- **expire_logs_days**：binlog 保留天数，建议至少 7 天
- **server-id**：服务器唯一标识，不能与其他 MySQL 实例重复

## 验证权限

### 检查用户权限

```sql
-- 查看用户权限
SHOW GRANTS FOR 'datatrace_sync'@'%';

-- 预期输出示例：
-- GRANT SELECT, SHOW VIEW ON `source_database`.* TO `datatrace_sync`@`%`
-- GRANT SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO `datatrace_sync`@`%`
```

### 检查 Binlog 配置

```sql
-- 检查 binlog 是否开启
SHOW VARIABLES LIKE 'log_bin';
-- 预期输出：ON

-- 检查 binlog 格式
SHOW VARIABLES LIKE 'binlog_format';
-- 预期输出：ROW

-- 查看 binlog 文件列表
SHOW BINARY LOGS;
```

## 安全建议

### 1. 最小权限原则

只授予必要的权限，不要授予 `INSERT`、`UPDATE`、`DELETE` 等写权限。

### 2. 限制访问来源

将 `%` 改为具体 IP 地址或 IP 段：

```sql
-- 限制为特定 IP
CREATE USER 'datatrace_sync'@'192.168.1.100' IDENTIFIED BY 'password';

-- 限制为 IP 段
CREATE USER 'datatrace_sync'@'192.168.1.%' IDENTIFIED BY 'password';
```

### 3. 使用强密码

- 密码长度 ≥ 16 位
- 包含大小写字母、数字、特殊字符
- 定期更换密码

### 4. SSL 加密连接

生产环境建议使用 SSL 加密连接：

```sql
-- 要求用户使用 SSL 连接
GRANT SELECT ON source_database.* TO 'datatrace_sync'@'%' REQUIRE SSL;
```

### 5. 定期审计

定期检查用户权限和访问日志：

```sql
-- 查看用户列表
SELECT user, host FROM mysql.user WHERE user = 'datatrace_sync';

-- 查看用户权限
SHOW GRANTS FOR 'datatrace_sync'@'%';
```

## 权限撤销

如果需要撤销权限或删除用户：

```sql
-- 撤销权限
REVOKE SELECT ON source_database.* FROM 'datatrace_sync'@'%';
REVOKE REPLICATION SLAVE, REPLICATION CLIENT ON *.* FROM 'datatrace_sync'@'%';

-- 删除用户
DROP USER 'datatrace_sync'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

## 常见问题

### Q1: 为什么需要 REPLICATION SLAVE 权限？

A: 增量同步需要读取 MySQL 的 binlog 来捕获数据变更。`REPLICATION SLAVE` 权限允许用户读取 binlog 事件。

### Q2: 这些权限会影响源数据库吗？

A: 不会。所有权限都是只读权限，不会对源数据库进行任何修改操作。

### Q3: 可以使用 root 用户吗？

A: 不推荐。建议创建专用的同步用户，遵循最小权限原则。

### Q4: 如何测试权限是否正确？

A: 在 DataTrace 的"数据源配置"页面，点击"测试连接"按钮，系统会自动检测权限是否满足要求。

### Q5: binlog_format 必须是 ROW 吗？

A: 是的。DataTrace 只支持 ROW 格式的 binlog，因为 ROW 格式包含完整的数据变更信息。

## 最小权限集合总结

### 仅全量同步

```sql
GRANT SELECT, SHOW VIEW, SHOW DATABASES ON *.* TO 'datatrace_sync'@'%';
```

### 全量 + 增量同步

```sql
GRANT SELECT, SHOW VIEW, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT 
ON *.* TO 'datatrace_sync'@'%';
```
