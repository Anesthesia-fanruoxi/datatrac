# DataTrace 设计概览

> 更新时间：2024年（重构后）
> 
> MySQL数据同步系统 - 简化架构版本

## 🎯 系统目标

构建一个高性能、易维护的MySQL数据同步系统，支持：
- ✅ 全量同步（快速、内存化）
- ✅ 增量同步（基于Binlog）
- ✅ 实时监控（SSE推送）
- ✅ 灵活配置（表映射、字段映射）

---

## 🏗️ 系统架构

### 技术栈

**后端**:
- Go 1.21+
- Gin (Web框架)
- GORM (ORM)
- go-mysql (Binlog解析)
- Redis (配置缓存 + 队列)

**前端**:
- 原生 JavaScript
- SSE (Server-Sent Events)
- 模块化设计

**存储**:
- MySQL (核心配置)
- Redis (详情配置 + 统计)
- 内存 (运行时进度)

## 📊 数据存储架构

```
┌─────────────────────────────────────────────────────────┐
│                      数据存储层                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  MySQL                Redis               内存          │
│  ├─ data_sources     ├─ task:config:*    ├─ 进度数据   │
│  └─ sync_tasks       └─ task:stats:*     └─ 临时状态   │
│                                                         │
│  持久化核心配置       缓存详情配置         运行时数据    │
│                      增量统计                           │
└─────────────────────────────────────────────────────────┘
```

### 存储职责

| 存储 | 内容 | 特点 |
|------|------|------|
| MySQL | 数据源、任务基本信息、最终状态 | 持久化、ACID |
| Redis | 表映射配置、增量统计 | 高性能、可持久化 |
| 内存 | 全量同步进度 | 极速、临时 |

## 🔄 同步模式

### 1. 全量同步模式

**流程**:
```
启动 → Redis读配置 → 内存进度 → 多Worker并发 → SSE推送 → 完成清空
```

**特点**:
- 完全在内存中维护进度
- 无数据库IO，性能极高
- 支持多线程并发
- 支持外键依赖排序
- 完成后清空内存

**适用场景**:
- 首次数据迁移
- 定期全量备份
- 数据重建

---

### 2. 增量同步模式

**流程**:
```
启动 → 捕获快照点
  ├─ 线程1: Binlog监听 → Redis队列（持续）
  └─ 线程2: 全量同步 → 内存进度 → 完成
       ↓
  增量消费 → Redis队列 → 目标库 → Redis统计 → SSE推送
```

**特点**:
- 双线程并行（全量 + Binlog监听）
- 全量阶段复用全量同步逻辑
- 增量阶段消费Redis队列
- 统计数据持久化到Redis

**适用场景**:
- 实时数据同步
- 数据库迁移后的持续同步
- 灾备系统

**配置向导**:
1. 选择数据源（源 + 目标）

3. 配置表映射（可选）

- 同步模式（full/incremental）

## 🔧 核心组件

### 1. ConfigCacheService

**职责**: Redis配置缓存管理

**方法**:
- `LoadTaskConfigToRedis()` - 加载配置到Redis
- `GetTaskConfigFromRedis()` - 从Redis读取配置
- `DeleteTaskConfigFromRedis()` - 删除Redis配置
- `InitAllTaskConfigs()` - 服务启动时初始化

---

### 2. TaskProgressManager

**职责**: 内存进度管理

**方法**:
- `InitTask()` - 初始化任务进度
- `UpdateUnitStatus()` - 更新单元状态
- `UpdateUnitProgress()` - 更新单元进度
- `GetTask()` - 获取任务进度
- `ClearTask()` - 清空任务进度

---

### 3. SyncEngine

**职责**: 全量同步引擎

**方法**:
- `InitializeWorker()` - 初始化阶段（创建表结构）
- `Worker()` - 数据同步Worker
- `SyncTable()` - 同步单个表

**特点**:
- 支持多Worker并发
- 支持外键依赖排序
- 进度实时更新到内存

---

### 4. IncrementalSync

**职责**: 增量同步总控

**组件**:
- BinlogListener - Binlog监听器
- BinlogQueue - 事件队列
- IncrementalConsumer - 增量消费者

**流程**:
1. 捕获快照点
2. 启动Binlog监听
3. 执行全量同步
4. 启动增量消费

---

## 📡 SSE 推送机制

### 推送端点

- `/api/tasks/:id/progress/stream` - 进度推送
- `/api/tasks/:id/logs/stream` - 日志推送
- `/api/tasks/:id/detail/stream` - 详情推送

### 推送频率

- 进度更新：每批次
- 日志推送：实时
- 详情更新：状态变化时

## 🎯 性能优化

### 1. 内存化进度

- 全量同步进度完全在内存
- 无数据库IO
- 极速更新

### 2. 多线程并发

- 支持配置Worker数量
- 外键表串行，普通表并发
- 充分利用CPU

### 3. 批量操作

- 批量读取数据
- 批量写入数据
- 减少网络往返

### 4. Redis缓存

- 配置缓存到Redis
- 减少MySQL查询
- 提高响应速度
